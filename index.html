Development

Quiz App

In this app, users are presented with a series of three questions. After they have been answered, the score and summary of choices are presented for review on the final screen.

Key Features
  • Question types include multiple-choice, multiple-answer, and true/false
  • Score is calculated and displayed
  • Summary of questions on final screen: choices are checked and highlighted green if correct; if not, the are grayed out and styled with a strikethrough; correct answers that aren't chosen are simply checked
Skills Highlighted
  • Expo is used for cross-platform tests
  • Employs React Native for UI components
  • Styles are utilized to better show whether the chosen answers were correct or incorrect

One problem I had while making this was that one of the questions had an answer already selected by default, which wasn't what I was intending. To fix this, I used "null" instead of "0" as that question's "useState()" value.

Creating this app gave me an understanding of how to incorporate various JavaScript logics and conditionals into React Native.

Exercise App

This exercise app allows users to choose from a list of exercises and track how much activity they've done through a rep counter or stopwatch.

Key Features
  • Repetition-based exercises include a rep counter
  • Duration-based exercises include a timer
  • Reset buttons for rep count/timer
  • A related exercise is suggested to the user, giving them the option to navigate to that exercise's screen from the previous exercise
Skills Highlighted
  • Expo is used for cross-platform tests
  • Employs React Native for UI components
  • JavaScript is used for stopwatch logic
  • Styles are utilized to create an attractive design and layout

While making this, I couldn't correctly implement the Suggested Exercise buttons. To make them work, I passed in the names of the suggested exercises as route params in the DurationExercise and RepetitonExercise functions and then used them in their respective navigation.navigate methods.

Developing this app gave me a better understanding of React Native and cross-platform testing.

To-Do App

In this simple to-do app, users can create tasks and check them off once completed.

Key Features
  • Users are able to add tasks to a compiled list
  • Tasks can be checked off or unchecked
  • Checked off tasks are styled with a strikethrough
Skills Highlighted
  • Expo is used for cross-platform tests
  • Employs React Native for UI components
  • Styles are utilized for better understanding of task status

During the creation process, I noticed that checking or unchecking a box would cause the whole list to disappear. I figured out that it was because I didn't add a rest parameter into the setTasks method in the update Task function.

This project was a foundational learning experience that helped me learn React Native. With it, I was able to craft a functional and simple app with real-world relevance and application.

Menu App

This online menu displays food items categorized into Dinner and Dessert sections.

Key Features
  • No hardcoded values, so items are able to be replaced with ease
  • All items include "Add to Cart" buttons
Skills Highlighted
  • React is utilized for UI components
  • Styles are utilized to gie a menu a more interesting look

Since I was very new to React when working on this, I missed putting in many important curly braces. These are there so that values don't have to be hardcoded and you can pass in JavaScript variables and objects containing whatever values you want.

This was a foundational activity crucial to my understanding of the React library. It also helped me learn how to code an online menu with options available for ordering items from it.

Flash Card App

FlashDeck would be beneficial to people who need to study and like to stay organized. The app would allow users to create decks of flashcards and flip through the cards to study them. Features would include:

  • creating, editing, and deleting cards and decks
  • studying the cards in sequence or shuffling them
  • color coding and titling decks
  • editable front and back of cards to separate prompt/question from answer

The color scheme gives the app a lighter and less daunting look, which is done to ease the user's possible stress surrounding studying. Orange is used as an accent color because it tends to represent confidence and optimism, both feelings that this app hopes to invoke.

Button shapes closely resemble ones used in other apps for familiarity's sake. Shuffling the cards makes the answers harder to predict. Color coding the decks helps users distinguish them better and gives the app a more fun feel.
